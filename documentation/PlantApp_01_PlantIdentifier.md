# PlantApp — 01. Определитель растений

## Обзор

Определитель растений — ключевая бесплатная фича PlantApp. У Planta определение за paywall ($36/год). У нас — бесплатно, с перекрёстной проверкой из нескольких источников.

## 1. Определение растения (Identification)

┌──────────────┐

## 2. Три категории растений

Определяются **биологией**, не намерением пользователя.

```
Все растения
├── Декоративные (decorative)
│   ├── Комнатные (indoor: true)
│   │   └── Монстера, фикус, кактус, орхидея, сансевиерия...
│   └── Уличные (indoor: false)
│       └── Берёза, дуб, ель, одуванчик...
│
├── Зеленные (greens)
│   └── Салат, шпинат, базилик, укроп, мята, петрушка...
│
└── Плодовые (fruiting)
    └── Томат, перец, клубника, мандарин, яблоня, огурец...
```

### Отличия категорий

| | Декоративные | Зеленные | Плодовые |
|---|---|---|---|
| Фазы роста | Нет | Нет | Да (рассада → вегетация → цветение → плодоношение) |
| Урожай | Нет | Листья/стебли | Плоды/ягоды |
| Спектр света | Статичный | Вегетативный (синий+белый) | Меняется по фазам (синий→красный) |
| Удобрение | Простое | Акцент на N (азот) | N→P/K по фазам |
| Карточка | Статичная | Статичная + урожай | С фазами |

### Дополнительные поля

```json
{
  "category": "fruiting",
  "indoor": true,
  "edible": true,
  "edible_parts": ["fruit"],
  "has_phases": true
}
```

Съедобность (`edible`) — отдельное поле, не категория. Декоративный перец: `category: "fruiting"`, `edible: false`.

## 3. Карточка растения

### Структура (принцип "Информация = Действие")

Порядок по приоритету: **выживание растения → безопасность → детали.**

**УРОВЕНЬ 1** (видит каждый, без скролла):

| # | Поле | Откуда | Примечание |
|---|------|--------|-----------|
| 1 | **Идентификация** | PlantNet/SerpAPI | Фото, научное + общее название, семейство |
| 2 | **Полив** | Perenual + preset | Частота, зимой, требовательность |
| 3 | **Свет + PPFD** | Perenual + литература | Предпочтительный, PPFD min/max, DLI ★ |
| 4 | **Токсичность** | Наша база (22 семейства) | Для животных, людей, примечание |
| 5 | **Проблемы и вредители** | Perenual | Типичные болезни, вредители |

**УРОВЕНЬ 2** (раскрыть "Подробнее"):

| # | Поле | Откуда |
|---|------|--------|
| 6 | **Температура** | Perenual | Диапазон, зимостойкость, сезоны на улице |
| 7 | **Влажность** | Perenual | Уровень, опрыскивание, min % |
| 8 | **Почва и пересадка** | Perenual | Типы почвы, частота |
| 9 | **Удобрение** | Perenual + литература | Тип, частота, сезон |
| 10 | **Характеристики** | Perenual | Размеры, жизненный цикл, сложность |

★ = уникальное, у Planta нет

### Три типа карточки

**Декоративные** — статичная, 10 пунктов как выше.

**Зеленные** — дополнительно:
- Спектр света (вегетативный: синий+белый)
- Удобрение с акцентом на азот (N)
- Блок "Сбор урожая" (когда, как)

**Плодовые** — дополнительно:
- Выбор фазы: `[Рассада ▾]` (пользователь выбирает)
- Полив, свет, удобрение **меняются по фазе**
- Спектр по фазе (синий → сбалансированный → красный)
- Блок "Сбор урожая"

## 4. База данных растений

### Принцип

Своя база, **независимая от сторонних сервисов**. Данные собираются заранее, не парсятся при каждом запросе. Растение определили → карточка уже готова → мгновенный ответ.

### Источники данных (разовый сбор)

| Источник | Бесплатно | Что даёт | Карта |
|---|---|---|---|
| **Perenual** | 100/день | Полив, свет, токсичность, почва, размеры, вредители | Нет |
| **Wikipedia** | Безлимит | Описание, фото (Wikimedia Commons), валидация | Нет |
| **Наша база** | Наше | 76 семейств → preset, start/stop %, токсичность | — |
| **Литература** | Наше | PPFD, DLI, спектры, фазы (668 МБ, 150+ документов) | — |

Принцип трёх источников: данные перекрёстно проверяются. Совпали 2-3 источника → истина.

### Хранение

**Решение: Turso (SQLite edge database)**

| Параметр | Значение |
|---|---|
| Технология | SQLite (стандарт, портативный) |
| Бесплатный тир | **9 ГБ** |
| Кредитная карта | Не нужна |
| 100K растений (~300 МБ) | С запасом (9 ГБ) |
| Рост до 500K+ | Хватит |
| Миграция | SQLite файл — копируется куда угодно |
| API | HTTP API из коробки |

Почему Turso, а не Supabase: Supabase = 500 МБ (впритык при 100K). Turso = 9 ГБ (запас в 30 раз). SQLite портативнее — один файл, можно перенести на любой сервер.

## 6. Референсы

### Существующий код (сайт)
- `polivalka-web/identify.html` — UI определителя
- `lambda/api_handler.py` → `identify_plant_handler()` — бэкенд (PlantNet)
- `lambda/api_handler.py` → `FAMILY_PRESETS` (76 семейств), `TOXIC_FAMILIES` (22 семейства)
- `lambda/api_handler.py` → `PRESET_DETAILS` (4 preset: succulents, standard, tropical, herbs)

### Литература (PPFD/спектры)
- `documentation/References/Literature/Гидропоника.../Formuly_dlya_perevoda_vattov_v_mikromoli.pdf` — LI-COR, формулы конверсии
- `documentation/References/Literature/Факторы роста растений/Свет/СПЕКТРЫ/СОЛНЦЕ/` — замеры Mulyarchik (спектрорадиометр)
- `documentation/References/Literature/Факторы роста растений/Свет/СПЕКТРЫ/` — спектры LED, фазы роста

### Конверсия Lux → PPFD (подтверждено LI-COR + Mulyarchik)
- Солнечный свет: `PPFD = Lux × 0.018` (1 klux = 18 µmol/m²/s)
- Белый LED: `PPFD = Lux × 0.012` (1 klux = 12 µmol/m²/s)
- Grow light (R+B): ненадёжно через люксы, нужен PAR-метр
