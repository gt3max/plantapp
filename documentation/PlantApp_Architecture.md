# PlantApp — Архитектура продукта

## Обзор

PlantApp — мобильное приложение для ухода за растениями с интеграцией Polivalka (автополив).
Три клиента (приложение + сайт + ESP32 web UI) работают на одном бэкенде (AWS Lambda + DynamoDB).

## Принципы продукта

1. **Цель: сохранить растение** — всё остальное вторично
2. **Информация = Действие** — показываем только то, что ведёт к действию
3. **Два уровня** — Level 1 (базовый, 90% пользователей), Level 2 (продвинутый)
4. **Честная монетизация** — бесплатно работает, Premium за офлайн и расширенное
5. **Try before you buy** — сначала попробуй, потом плати
6. **Три источника истины** — перекрёстная проверка данных
7. **Независимость** — своя база, без vendor lock-in

## Три категории растений

```
Все растения
├── Декоративные (без плодов, без урожая)
│   ├── Комнатные: монстера, фикус, кактус, орхидея...
│   └── Уличные: берёза, дуб, ель...
│
├── Зеленные (урожай = листья/стебли)
│   └── Салат, шпинат, базилик, укроп, мята...
│
└── Плодовые (есть плоды → есть фазы)
    └── Томат, перец, клубника, мандарин, яблоня...
```

Категория определяется **биологией растения**, не пользователем.

## Конкуренты

| | Planta | PlantApp |
|---|---|---|
| Определение | Платное ($36/год) | Бесплатное |
| База видов | ~25,000 | Цель 100K+ |
| Свет | Люксы (4 кнопки) | PPFD + DLI (фотоны) |
| Реальные данные | Нет (алгоритмы) | Сенсор Polivalka |
| Офлайн | Не работает | Premium: полная база |
| Автополив | Нет | Polivalka |
| Монетизация | Агрессивный paywall | Честная, try before you buy |

## Стек

- **Приложение:** React Native + Expo (SDK 54)
- **Бэкенд:** AWS Lambda + API Gateway + DynamoDB (существующий)
- **База растений:** Turso (SQLite edge, 9 ГБ бесплатно) + бэкап в GitHub (JSON по семействам)
- **Идентификация:** PlantNet (primary) + SerpAPI Google Lens (secondary)
- **Данные:** Perenual + Wikipedia + своя литература (668 МБ)
